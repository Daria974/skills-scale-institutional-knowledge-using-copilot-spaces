# OctoAcme Project Management Docs

OctoAcme runs projects through a lightweight lifecycle: **Initiation → Planning → Execution → Release → Close/Retrospective**. Work begins with a Project One-pager/Charter to align on the problem, SMART objective, success metrics, stakeholders, timeline, and initial risks before a go/no-go decision.

After approval, the team plans into a prioritized backlog with clear acceptance criteria and a shared **Definition of Done (DoD)**. Delivery then follows a board-based workflow (**Backlog, Ready, In Progress, In Review, QA, Done**) with small PRs linked to issues, CI checks, and team-defined approvals.

Core roles include the **Project Manager (PM)**, **Product Manager (PdM/Product Lead)**, **Developers**, **QA/Testing**, and **Stakeholders**. Communication is structured through daily standups, a weekly delivery sync, sprint/milestone demos, and regular stakeholder updates, with escalation from team triage to PM/Product Lead to sponsor-level support when needed.

Quality and release practices are integrated into day-to-day work: unit/integration coverage, critical-flow smoke testing, CI lint/test/security scanning, release notes, rollback planning, post-deploy verification, and retrospectives that convert learnings into owned, time-bound improvements.
