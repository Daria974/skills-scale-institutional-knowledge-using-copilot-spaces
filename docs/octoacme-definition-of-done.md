# OctoAcme — Definition of Done (DoD)

Use this baseline DoD for backlog items, features, and release candidates.

## Baseline DoD
- Scope is complete and mapped to acceptance criteria.
- Code is reviewed and approved per team policy.
- CI checks pass (lint, tests, and security scanning as configured).
- Required tests are added/updated (unit/integration; smoke tests for critical flows).
- User-facing and operational docs are updated where applicable.
- Feature flags/config changes are documented.

## Release Quality Gates
- Release notes drafted and validated.
- Rollback plan is documented and tested where practical.
- Monitoring/alerts are in place for impacted areas.
- Post-deploy verification checklist is completed.

## Exceptions
- Any waived DoD item must be explicitly documented with:
  - reason,
  - approver,
  - follow-up task and due date.
